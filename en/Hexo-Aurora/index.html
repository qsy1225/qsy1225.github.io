Getting Started & Declaration

Some friends want to remove the “Powered by Hexo & Themed by Aurora” footer when using the Aurora theme. However, the official documentation for Hexo Aurora doesn’t mention footer customization, and there’s no footer section in the config.aurora.yml file.

Here’s how to do it.

Disclaimer:
I’m not a professional, so there may be inaccuracies in this tutorial. Please proceed with caution.

This guide is based on Aurora v2.5.3 installed via npm.
Back up your site before making changes!
If you encounter issues, feel free to comment—I’ll help if I can.

Locating the Relevant Files

Original Footer:

  1. Navigate to your blog’s root directory.

  2. Go to: node_modules/hexo-theme-aurora/layout/index.ejs and open the file.

  3. Look for the script tag. The filename (e.g., 120aa8f8.js) may vary but follows the same pattern:

  4. Return to the parent directory and navigate to source/static/js/.

  5. Right-click the JavaScript file (e.g., 120aa8f8.js) and open it with a text editor (Notepad, Sublime Text, etc.).

  6. Use Ctrl+F (or Edit > Find) to search for Powered. You’ll find the relevant code:

Modifications

1. Remove Only “Themed by Aurora”

Delete the marked code:

,v("a",H6,[v("b",U6," Aurora v"+K(e.themeConfig.version),1)])

Result:

2. Remove Both “Powered by” and “Themed by”

Delete the entire line:

,v("li",null,[Ve(" Powered by "),Z6,Ve(" & Themed by "),v("a",H6,[v("b",U6," Aurora v"+K(e.themeConfig.version),1)])])

Result:

Save the file after editing, then run:

hexo cl
hexo g
hexo s

Check locally. If everything works, deploy with hexo d.

Critical Note:
Ensure you remove the correct commas (,). Leaving extra commas may cause infinite page loading or layout issues!

3. Line Break for “All Rights Reserved”

To move “All Rights Reserved” to a new line:

Original Code:

Ve(" . All Rights Reserved. ")

Modified Code:

Ve("."),v("br"),Ve(" All Rights Reserved. ")

Effect:

Add a space if the period appears too close:

Ve(". "),v("br"),Ve("All Rights Reserved. ")

Effect:
